# data_analysis — 引导式数据分析 Skill

> 帮助你从一句模糊的"帮我分析一下"出发，一步步厘清目标、理解数据、建立指标体系，最终产出分析报告和可视化页面。

---

## 安装

**安装 skill：**

```bash
# 下载仓库
git clone https://github.com/ldy-123/dy_skills.git

# 将整个 skill 文件夹复制到你的 agent 的 skill 目录
cp -r dy_skills/data_analysis <你的 agent skill 目录>/
```

安装指令有报错，可以直接截图随便问一个AI

## 为什么需要这个 Skill

普通的数据分析 AI 默认你已经知道自己想分析什么。但大多数人并不清楚：

- 我真正想回答的问题是什么？
- 这些字段到底代表什么业务含义？
- 该看哪些指标？

这个 Skill 把"发现分析目标"这个过程也纳入进来，让 AI 主动承担信息差填补的工作，你只需要确认或纠错。

---

## 使用方法

1. 把 `SKILL.md` 放到你的项目文件夹里
2. 打开 agent，直接说：

```
帮我分析这份数据
```

也可以把数据文件和分析背景一起发给 AI，任意一步提供都可以。

**支持的数据格式**：xlsx、csv、xls、json

每一步都可以跳过，直接说"跳过这步"即可。

---

## 六步工作流

| 步骤 | 做什么 | 产出 |
|------|--------|------|
| STEP 1 | 挖掘分析背景，把模糊需求变成明确目标 | 分析目标确认 |
| STEP 2 | 补齐业务信息差，逐字段确认数据含义 | `field_definitions.md` |
| STEP 3 | 构建指标体系，用户确认后再开始分析 | 指标体系框架 |
| STEP 4 | 执行分析，主动发现额外规律 | 分析结论 |
| STEP 5 | 双格式输出报告 | `analysis_report.md` + `analysis_report.html` |
| STEP 6 | 把本次背景和指标体系固化成可复用 Skill | `[场景]-analysis/` 文件夹 |

---

## 产出文件

- **`field_definitions.md`**：字段定义确认表，AI 推测 + 你确认
- **`analysis_report.md`**：完整分析报告，可存档和二次编辑
- **`analysis_report.html`**：自包含可视化页面，可直接分享，无需安装任何依赖
- **`[场景]-analysis/`**：专属 Skill 文件夹，下次同类数据直接跳过所有访谈步骤

---

## STEP 6：生成专属 Skill（可选）

分析完成后，AI 会询问是否生成专属 Skill。生成的文件夹结构：

```
[场景]-analysis/
├── SKILL.md              # 执行逻辑 + references 文件引用路径
└── references/
    ├── business_context.md   # 业务背景
    ├── field_definitions.md  # 字段定义
    └── metrics.md            # 指标体系（按需）
```

生成后，下次上传同类数据只需说"帮我分析这份数据"，AI 会跳过所有访谈，直接按本次建立的指标体系跑分析。

如果数据结构或分析目标发生变化，可以让 AI 根据变化修改 skill 内容（推荐），或者重新跑一次完整流程生成新的 Skill。

---

## 数据不足时的处理

遇到以下情况，AI 会停下来告知你，并询问是否继续或补充数据：

- 数据不足 30 行
- 关键字段缺失率超过 20%
- 分析目标所需字段不存在
- 数据时间跨度过短（如仅单天）
